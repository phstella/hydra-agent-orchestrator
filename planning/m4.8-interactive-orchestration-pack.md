# M4.8 Local Execution Pack (Interactive Orchestration Console)

Last updated: 2026-02-25

## 1. Purpose

This is the local-first execution tracker for `M4.8`.

`M4.8` targets the interactive orchestration-console experience:
- multiple concurrent interactive sessions
- multiple sessions of the same adapter type
- mockup-aligned operator flow (create panel + focused output + active lanes rail)

This milestone is explicitly separate from race/scoring behavior.

## 2. Locked Decisions

1. Interactive orchestration and race orchestration are separate feature tracks.
2. Interactive lane identity is session-based (`session_id`), not adapter-key-based.
3. Duplicate adapter sessions are first-class (for example two `codex` lanes at once).
4. Race IPC/scoring/merge semantics are out of scope and must not change.
5. Collaboration workflows remain Phase 5 scope and are out of scope for this milestone.

## 3. Completion Gate

`M4.8` is complete only when:
1. Operator can spawn/manage N concurrent interactive sessions from one console.
2. Operator can spawn multiple sessions of the same adapter type.
3. Session focus, polling, and interventions are lane-scoped and isolated.
4. Existing race behavior is regression-free.

## 5. Out of Scope

1. Workflow DAG/presets (Phase 5).
2. Cross-agent collaboration semantics (review/refine loops, specialization handoff).
3. Multi-user/live-shared sessions.
4. Auto-merge or score-driven autonomous decisions.
