# P4.9.8 Multi-Workspace Thread Orchestration Pack

Last updated: 2026-03-09

Purpose: track implementation progress for expanding orchestration so users can create threads from any folder, while creating a new git worktree when launching an additional thread for the same source folder.

## Scope

1. Allow per-thread folder selection (not tied to one global workspace only).
2. Support multiple threads targeting the same source folder.
3. When a same-folder thread already exists, launch the new thread in a dedicated worktree.
4. Preserve existing safety/capability gates and lifecycle cleanup guarantees.

## Current State Snapshot

1. Interactive start path already accepts `cwd`, resolves repo root, and can auto-init git for non-repo folders.
2. Interactive start path does **not** allocate worktrees today; concurrent threads for one repo share the same checkout.
3. Worktree lifecycle service exists and is production-used by race mode (`hydra-core::worktree::WorktreeService`).
4. Interactive session metadata/IPC does not expose source/repo/worktree paths.
5. Orchestration UI still assumes a single workspace context for thread creation.
